# 飞牛 fnOS 原生应用构建计划

## 目标
为电表数据管理系统创建飞牛 fnOS 原生应用包（.fpk），通过飞牛应用中心安装，无需 Docker。

## 原理
项目已使用 Next.js standalone 模式构建（`.next/standalone/server.js`），可直接在飞牛 NAS 的 Node.js 环境中运行。只需创建飞牛应用包结构，将构建产物放入，并编写生命周期管理脚本。

## 需要创建的文件

所有文件位于 `fnos/App.Native.ElecMeter/` 目录下：

### 1. `manifest` - 应用元数据
```
appname=App.Native.ElecMeter
version=1.0.0
display_name=电表数据管理
desc=电表数据管理系统，支持手动录入电表读数、自动计算用电量、数据可视化分析。
platform=all
source=thirdparty
maintainer=elec-dev
distributor=elec-dev
os_min_version=0.9.27
desktop_uidir=ui
desktop_applaunchname=App.Native.ElecMeter.Application
service_port=16543
```

### 2. `cmd/main` - 生命周期管理脚本
基于飞牛 Native 模板，使用 PID 文件管理 Node.js 进程：
- `start`: 启动 `node server.js`，PID 写入 `$TRIM_PKGVAR/app.pid`
- `stop`: 发送 SIGTERM 优雅关闭
- `status`: 检查 PID 是否存活
- 日志输出到 `$TRIM_PKGVAR/info.log`
- 数据库路径使用 `$TRIM_PKGVAR/elec.db`（通过环境变量覆盖）

### 3. `app/server/` - 构建产物目录
从 `.next/standalone/` 复制：
- `server.js` - Next.js standalone 服务器
- `node_modules/` - 依赖
- `.next/` - Next.js 构建产物

### 4. `app/ui/config` - 应用入口配置
```json
{
    ".url": {
        "App.Native.ElecMeter.Application": {
            "title": "电表数据管理",
            "icon": "images/icon_{0}.png",
            "type": "url",
            "protocol": "http",
            "port": "16543"
        }
    }
}
```

### 5. `app/ui/images/icon_64.png` + `icon_256.png` - 占位图标
使用 PowerShell 生成简单纯色占位图标

### 6. `config/privilege` - 权限配置
默认 package 用户模式

### 7. `config/resource` - 资源配置
声明 data-share 用于数据持久化

### 8. `wizard/install` - 安装向导
端口配置（默认 16543）

### 9. `wizard/uninstall` - 卸载向导
数据保留选项

### 10. `ICON.PNG` + `ICON_256.PNG` - 包图标

## 构建脚本

创建 `fnos/build.sh`：
1. 执行 `npm run build` 构建 Next.js 项目
2. 清理并创建 `fnos/App.Native.ElecMeter/app/server/` 目录
3. 复制 `.next/standalone/*` 到 `app/server/`
4. 复制 `.next/static` 到 `app/server/.next/static`
5. 复制 `public/` 到 `app/server/public/`
6. 创建数据目录 `app/server/data/`
7. 下载 fnpack 工具（如未安装）
8. 执行 `fnpack build` 生成 .fpk 文件

## 关键环境变量（飞牛系统提供）
- `$TRIM_APPDEST` - 应用安装目录（含 app/ 内容）
- `$TRIM_PKGVAR` - 运行时数据目录（数据库存放处）
- `$TRIM_PKGETC` - 配置文件目录
- `$TRIM_SERVICE_PORT` - 服务端口

## 需要修改的现有文件

### `src/lib/db.ts`
- 数据库路径改为读取环境变量 `ELEC_DB_PATH`，默认 `data/elec.db`
- 这样在飞牛环境中可通过 `$TRIM_PKGVAR/elec.db` 指定持久化路径
